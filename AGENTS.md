# AGENTS.md — Erulean Tools (React 18 / TypeScript / Vite)

A browser-based editor for the Windrose Saga game data. It loads a
`game-data.json` file into an in-browser store, lets you create and edit game
entities (units, actions, augments, items, dungeon prefabs, level classes, and
ordered vocabularies), and exports the result back out as `game-data.json`.
Deployed to GitHub Pages at <https://windrose-saga.github.io/erulean-tools/>.

This repo is one of two children in the `erulean-metarepo` (alongside
`windrose-saga`, the Godot game). The Godot repo's `data/game-data.json` is the
**authoritative** game data; the `game-data.json` checked in here is sample/test
data used when `VITE_USE_TEST_DATA` is set. Round-trip exported files back into
`windrose-saga/data/game-data.json`.

## Build & Lint

- **Package manager:** `yarn` (v1.22.22, pinned via `packageManager`). Use
  `yarn install --frozen-lockfile` to match CI.
- **Dev server:** `yarn dev` (Vite on port 3001).
- **Build:** `yarn build` (runs `tsc` then `vite build`).
- **Lint:** `yarn lint` (ESLint, `--max-warnings 0` — warnings fail CI).
- **Format:** `yarn format:check` / `yarn format:fix` (Prettier).
- **Tests (final verification):** `yarn test` (Vitest, run once). Use
  `yarn test:watch` while iterating. Tests run in a `jsdom` environment because
  the ingest pipeline touches `localStorage` (zustand `persist`) and
  `import.meta.env`.
- **Deploy:** `yarn deploy` (builds, then publishes `dist/` to GitHub Pages via
  `gh-pages`). CI also auto-deploys on push to `main`/`master`.
- **Pre-commit:** Husky + lint-staged run on staged files; do not bypass.
- **CI** (`.github/workflows/ci.yml`) runs lint → build → test on every PR.
  Keep all three green before handing off.
- **Env:** Copy `example.env` to `.env`. `VITE_USE_TEST_DATA=true` auto-loads the
  bundled `game-data.json`; comment it out to start from an uploaded file.

## Architecture

- **src/routes/**: File-based TanStack Router routes. Naming maps to URLs:
  `entity.index.tsx` (list), `entity.new.tsx` (create), `entity.$id.tsx`
  (detail/edit). `__root.tsx` is the layout shell.
- **src/routeTree.gen.ts**: Auto-generated by the TanStack Router Vite plugin.
  **Do not edit by hand**; it regenerates on dev/build.
- **src/screens/**: Top-level pages not tied to an entity CRUD route
  (`Upload.tsx`, `DamageCalculator.tsx`).
- **src/store/**: Zustand store. `useGameStore.ts` is the single source of truth
  (zustand + `persist` + `immer`), holding entities keyed by id plus id↔name
  maps and ordered vocabularies. `getters/` expose typed selector hooks per
  entity. Selectors are auto-generated via `createSelectors` (`useGameStore.use.*`).
- **src/components/**: Presentational + container components. `forms/` holds the
  react-hook-form entity forms; the rest are lists, detail views, selects, and
  field arrays.
- **src/types/**: TypeScript models for each entity and the top-level
  `GameData` shape (the import/export contract).
- **src/constants/**: `DEFAULT_*` seed objects used when creating new entities.
- **src/utils/**: Ingest/export pipeline (`useIngest`, `useExportStore`,
  `validateIngest`, `generate*IdsMap`), vocab id helpers (`vocabId`), env, and
  generic hooks/helpers.

### Data flow

1. **Ingest** (`useIngest` / `useTestData`): parse a `game-data.json`, validate
   (`validateIngest`), merge with `DEFAULT_*` constants, build id maps, and load
   into `useGameStore`.
2. **Edit**: routes/forms read and mutate the store through getter hooks and
   store actions.
3. **Export** (`useExportStore`): reassemble the store back into the `GameData`
   shape and download it.

### Vocabularies (loot categories, generator tags)

These are **append-only ordered lists** because each entry's index is a Godot
enum ordinal consumed by `windrose-saga`. Removed entries are *tombstoned*
(`removed*Ids`) — hidden from the UI and stripped from items/units, but never
spliced out, so existing ordinals stay stable. A vocab id must be a valid
GDScript enum member name: `^[A-Z_][A-Z0-9_]*$` (see `utils/vocabId.ts`). Use
`normalizeVocabId` to coerce input, then `isValidVocabId` to validate.

## Code Style (TypeScript / React)

- **Formatting (Prettier):** single quotes, semicolons, trailing commas
  (`all`), 2-space indent, 100-char print width. Run `yarn format:fix`.
- **Linting:** ESLint airbnb + airbnb/hooks + airbnb-typescript + prettier, with
  type-checked rules. `import/order` is enforced: groups `[builtin/external] →
  internal → sibling → index`, alphabetized, newlines between groups. Let the
  linter sort imports.
- **Components:** Function components, typed (`React.FC` or explicit prop
  types). Hooks-first; respect `react-hooks/recommended`.
- **State:** Read/write the store only through `useGameStore.use.*` selectors
  and store actions — don't reach into store internals. Use `immer`-style
  mutations inside store actions.
- **Forms:** Build entity forms with `react-hook-form`; seed new entities from
  the matching `constants/DEFAULT_*`.
- **Types:** Prefer explicit types for entities and the `GameData` contract.
  `no-explicit-any` is off, but avoid `any` in new code; some `no-unsafe-*`
  rules are relaxed for the ingest pipeline only.
- **Tests:** Co-locate `*.test.ts(x)` next to the code under test (Vitest +
  jsdom). The ingest/validate/vocab pipeline is the most test-covered area —
  add tests there for any new validation or transform logic.
